## Оценка TC-001

**Статус: BLOCKED**

---

### Анализ предоставленного evidence

TC-001 содержит **визуальный критерий**: «отображается корректно с иконкой и текстом» — формулировка описывает то, как элемент выглядит пользователю (наличие иконки, текста, корректность отображения).

Предоставленное evidence: `querySelector вернул элемент` — подтверждает только **существование элемента в DOM**, но не его визуальный рендеринг.

### Почему этого недостаточно

Согласно **Принципу 8 (Visual TC = Screenshot + Self-Review)**:

- `querySelector` / DOM-assertion подтверждает, что элемент присутствует в модели данных
- Между DOM и рендерингом лежит CSS — DOM-assertion его не видит
- `button[icon]` в DOM ≠ «иконка визуально отображается»: элемент может быть скрыт (`display:none`), иконка может не загрузиться, текст может быть обрезан overflow

### Что требуется для PASS

| Шаг | Действие |
|-----|----------|
| 1 | Сделать скриншот кнопки |
| 2 | Открыть скриншот через Read и описать видимое одним предложением |
| 3 | Убедиться, что иконка и текст визуально присутствуют и не обрезаны |
| 4 | Сохранить скриншот в `reports/` |

### Зафиксированный результат

```
TC-001: Кнопка отображается корректно с иконкой и текстом
Статус: BLOCKED
Причина: Evidence недостаточен для визуального TC.
  - Предоставлено: querySelector вернул элемент (DOM-assertion)
  - Требуется: скриншот + визуальная проверка наблюдаемого рендеринга
  - querySelector подтверждает существование в DOM, но не визуальное
    отображение иконки и текста в реальной среде исполнения
```

Для закрытия тикета необходимо выполнить реальную визуальную проверку через браузер/desktop-инструмент и приложить скриншот с описанием наблюдаемого.
