## TC-001 — Фиксация итогового статуса

---

### Статус: `BLOCKED`

**Причина блокировки: недостаточный evidence для визуального тест-кейса**

---

### Анализ

**TC-001 содержит визуальный критерий:**
> «Кнопка отображается корректно с иконкой и текстом»

Формулировка содержит «отображается корректно», «с иконкой», «с текстом» — это описание того, **как элемент выглядит пользователю**. Согласно **Принципу 8** скила, это визуальный TC.

**Предоставленный evidence: `querySelector вернул элемент`**

| Что проверяет | Результат |
|---------------|-----------|
| DOM-присутствие элемента | ✓ подтверждено |
| Иконка визуально отрисована | ✗ **не проверено** |
| Текст кнопки видим и читаем | ✗ **не проверено** |
| Корректное визуальное оформление | ✗ **не проверено** |

`querySelector` подтверждает **существование элемента в DOM-модели**, но не его визуальный рендеринг. Между DOM и тем, что видит пользователь, лежит CSS — который DOM-assertion не видит. Иконка может не загрузиться, текст может быть скрыт через `visibility: hidden` или `color: transparent`, элемент может быть за viewport.

**Согласно Принципу 8:** для визуального TC обязательны:
1. Скриншот проверяемого элемента, сохранённый в `reports/`
2. Открытие скриншота через Read и визуальная самопроверка **до** записи PASS
3. В evidence: ссылка на PNG + описание того, что на нём видно

---

### Требуемые действия для разблокировки

Провести тестирование повторно с корректной методологией:

```
1. Открыть страницу/экран с кнопкой через Playwright MCP (веб) или Windows-MCP (desktop)
2. Сделать скриншот кнопки → сохранить в reports/TC-001-screenshot.png
3. Открыть скриншот через Read → описать увиденное (иконка присутствует? текст читаем? оформление корректно?)
4. Записать PASS только если визуально подтверждено соответствие критерию
```

---

**Итог:** DOM-assertion (`querySelector вернул элемент`) — тавтологична для визуального TC. Этот evidence не даёт нового подтверждения и не может быть основанием для PASS/FAIL. TC остаётся непроверенным по существу.
