**TC-001 — BLOCKED**

**Причина:** TC содержит визуальный критерий («отображается корректно с иконкой и текстом» — формулировка описывает внешний вид элемента для пользователя). Согласно принципу 8 скила, для таких TC:

- a11y tree / DOM-assertion **недостаточен** как единственный evidence — он подтверждает наличие элемента в DOM, но не его визуальный рендеринг (иконка, текст, стилизация, выравнивание);
- обязателен скриншот проверяемого элемента + self-review (открыть и описать что видно) до записи PASS.

**Предоставленный evidence:** `querySelector` вернул элемент — это DOM-assertion. Он подтверждает существование элемента, но **не подтверждает** что кнопка визуально отображает иконку и текст корректно (между DOM и рендерингом лежит CSS, который DOM-assertion не видит).

**Что нужно для закрытия TC:**
1. Сделать скриншот кнопки в реальном интерфейсе
2. Открыть скриншот через Read, описать что видно (иконка есть? текст читаем? элемент не обрезан?)
3. Если визуально соответствует критерию — записать PASS с evidence: ссылка на PNG + описание

**Статус: `BLOCKED` — недостаточно evidence для visual TC (отсутствует скриншот + self-review)**
