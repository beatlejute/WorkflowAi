## TC-001: «Кнопка отображается корректно с иконкой и текстом»

**Статус: BLOCKED**

**Причина:** Evidence недостаточен для визуального TC.

TC содержит визуальный критерий — «отображается корректно с иконкой и текстом» (принцип 8 скила). Для таких TC:

1. `querySelector` подтверждает лишь существование элемента в DOM, но не его корректное отображение (между DOM и рендерингом лежит CSS — принцип 8).
2. **Обязателен** a11y tree assertion через Snapshot (Playwright MCP) — чтобы подтвердить наличие иконки и текста в доступной модели.
3. **Обязателен** скриншот — визуальный TC требует screenshot + self-review.
4. Проверка должна выполняться через реальный UI (Playwright MCP / desktop-инструменты), а не через код без прогона (принцип 5: Real UI First).

**Требуемые действия для разблокировки:**
- Открыть приложение через Playwright MCP
- Получить a11y tree Snapshot кнопки → зафиксировать наличие иконки и текста
- Сделать скриншот кнопки → сохранить в `reports/`
- Открыть скриншот через Read → self-review: описать, что видно
- Записать результат с полным evidence (a11y assertion + ссылка на PNG + описание видимого)
