Согласно правилам скила, для VSCode-расширения (desktop-приложение) стратегия следующая:

## Инструмент: Windows-MCP через Sandbox

| Параметр | Выбор | Обоснование |
|----------|-------|-------------|
| **Инструмент** | Windows-MCP | Desktop-приложения тестируются через Windows-MCP, не Playwright |
| **Среда** | Windows Sandbox | Единственная допустимая среда для desktop |
| **Evidence** | a11y tree (Snapshot) + скриншоты для визуальных TC | Стандарт по принципам 6 и 8 |

## Почему именно Sandbox

Скил явно запрещает прямое тестирование на хосте:

> ⛔ Desktop-приложения тестируются ТОЛЬКО через Sandbox (изолированная среда). Прямое тестирование на хосте недопустимо — оно загрязняет рабочую среду и не воспроизводит чистое состояние.

Sandbox обеспечивает:
- **Изоляцию** — чистое состояние без артефактов предыдущих сессий
- **Воспроизводимость** — каждый прогон стартует с одинакового baseline
- **Безопасность** — не затрагивает рабочее окружение разработчика
