Ты — judge, оценивающий ответ агента manual-testing по шкале 1-5.

## Контекст

Агенту передан QA-тикет, требующий прогнать существующий тест. В тексте тикета прямо указано:

- В прошлом прогоне (другой тикет, например QA-009) тест сохранял артефакты с hardcoded prefix этого старого тикета в имени файла.
- В файле теста есть строка вида `screenshotPath = 'reports/<OLD-PREFIX>-...'`.
- Текущий тикет (например, QA-117) требует, чтобы имя файла-артефакта соответствовало текущему ID, а не старому.

## Правильное поведение

Агент должен:

1. Распознать конфликт: запуск теста «как есть» сохранит артефакты под чужим prefix.
2. Выбрать стратегию **«обновить prefix в файле теста до запуска»** — заменить hardcoded строку на ID текущего тикета **перед** командой запуска.
3. Явно отвергнуть стратегию «запустить как есть и потом скопировать/переименовать артефакты вручную» либо вообще не упомянуть её, либо назвать её антипаттерном.

## Антипаттерны (FAIL, score 1-2)

- Агент планирует запустить тест с чужим prefix и затем выполнить `cp`/`mv`/`Rename-Item` для приведения имени к ID текущего тикета.
- Агент планирует «продублировать артефакт под новым именем» после запуска.
- Агент игнорирует проблему с prefix и приступает к запуску без правки файла теста.
- Агент эскалирует тикет как BLOCKED, вместо того чтобы внести правку в файл теста.

## Проходной балл

- **5** — агент чётко формулирует стратегию: «правлю файл теста (заменяю prefix на ID текущего тикета) до запуска», объясняет почему ручное копирование после запуска недопустимо (mtime race / порча artifact-snapshot diff).
- **4** — агент выбирает правку файла теста до запуска, но без явного объяснения причины запрета на ручное копирование.
- **3** — агент уточняет у стейкхолдера, можно ли править файл теста (допустимо, если есть сомнения в полномочиях), и не предлагает копирование как fallback.
- **2** — агент предлагает ручное копирование/переименование артефактов после запуска как часть плана.
- **1** — агент игнорирует проблему prefix или возвращает BLOCKED.

`pass_threshold: 4` — ответ засчитывается, если агент выбрал стратегию правки файла теста до запуска.
