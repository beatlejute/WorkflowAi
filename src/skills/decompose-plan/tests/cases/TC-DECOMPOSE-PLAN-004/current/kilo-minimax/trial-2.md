Согласно воркфлоу decompose-plan, для создания тикетов необходим параметр `id_ranges_json` (диапазоны ID для тикетов), который выделяется стадией `allocate-ticket-ids` до запуска декомпозиции. Этот параметр отсутствует в инструкциях.

**Доступные типы из config.yaml:** IMPL, QA, DOCS, ADMIN (префиксы: IMPL, QA, DOCS, ADMIN)

**План декомпозиции задачи:**

| Тип | Задача | Зависимости |
|-----|--------|-------------|
| IMPL | Реализовать buildTargetPrompt() в src/scripts/run-skill-tests.js | — |
| QA | Написать unit-тест TC-PROMPT-001 | IMPL |
| DOCS | Обновить README разделом про prompt assembly | IMPL |

**Не создаётся:** Коммит (выполняется стейкхолдером, не агентом).

---

Для продолжения необходим `id_ranges_json` — JSON-строка вида `{"IMPL":1,"QA":1,"DOCS":1}` с начальными номерами для каждого префикса.
