# Типы задач

Справочник типов задач, их особенностей и подходов к выполнению.

## Основные типы

| Тип | Описание | Основное действие | Воркфлоу |
|-----|----------|-------------------|----------|
| `impl` | Реализация новой функциональности | Написать код | `workflows/execute.md` |
| `fix` | Исправление бага | Найти причину → исправить → проверить | `workflows/execute.md` |
| `docs` | Документация | Написать/обновить документы | `workflows/execute.md` |
| `review` | Ревью кода/документов | Проверить → зафиксировать замечания | `workflows/execute.md` |
| `plan` | Создание плана | Составить план (только рекомендации) | `workflows/execute.md` |
| `admin` | Настройка/конфигурация | Read → Edit/Write → Verify | `workflows/admin.md` |
| `rsh` | Исследование | Запросы к API → синтез → отчёт | `workflows/research.md` |
| `coach` | Улучшение скилов | Аудит/создание/улучшение скила | Определяется скилом коуча |
| `arch` | Архитектурное решение | Анализ → проектирование → документирование | `workflows/execute.md` |

## Детали по типам

### IMPL — реализация

- Следовать методологиям TDD, SOLID, DRY
- Если есть тесты — запустить после изменений
- Если нет тестов — написать базовые
- Не вносить изменения за пределами scope тикета

### FIX — исправление бага

- Сначала воспроизвести проблему (если возможно)
- Найти корневую причину (не патчить симптомы)
- Написать тест на баг (если возможно)
- Проверить, что исправление не ломает другое

### DOCS — документация

- Проверить актуальность существующей документации
- Использовать единый стиль проекта
- Включить примеры где уместно

### REVIEW — ревью

- Проверить соответствие DoD
- Зафиксировать замечания структурированно
- Оценить: код ✓, тесты ✓, документация ✓, безопасность ✓

### ADMIN — настройка

- **Обязательно** реальное изменение файлов (Read → Edit → Verify)
- Нельзя завершить без фактических правок
- Подробный чеклист — см. `workflows/admin.md`

### RSH — исследование

- **Обязательно** использовать скрипт исследований (не только собственные знания)
- Данные должны быть подкреплены источниками
- Подробный процесс — см. `workflows/research.md`

## Capabilities (требования к исполнителю)

| Capability | Описание |
|------------|----------|
| `code_generation` | Генерация кода |
| `code_editing` | Редактирование кода |
| `file_operations` | Работа с файлами |
| `debugging` | Отладка |
| `code_analysis` | Анализ кода |
| `documentation` | Написание документации |
| `strategic_thinking` | Стратегическое планирование |
| `typescript` | Знание TypeScript |
| `python` | Знание Python |

## Complexity (сложность)

| Уровень | Описание | Типичный объём |
|---------|----------|----------------|
| `simple` | Одно действие, минимальный контекст | 1-2 файла |
| `medium` | Несколько шагов, средний контекст | 3-5 файлов |
| `complex` | Много шагов, широкий контекст, архитектурные решения | 5+ файлов |
