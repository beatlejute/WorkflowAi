# Execute-task

Скил исполнения задач из тикетов workflow. Берёт тикет из `in-progress/`, выполняет работу и записывает результат.

## Структура

```
execute-task/
├── SKILL.md                    # Ядро: роль, ограничения, маршрутизация, принципы
├── README.md                   # Этот файл
├── workflows/
│   ├── execute.md              # Стандартное выполнение (impl, fix, docs, review, plan, arch)
│   ├── admin.md                # Admin-задачи (Read → Edit → Verify)
│   └── research.md             # Research-задачи (RSH) через скрипт исследований
└── knowledge/
    ├── ticket-structure.md     # Структура тикета, поля и их семантика
    └── task-types.md           # Типы задач, capabilities, complexity
```

## Как это работает

1. **Читает тикет** из `in-progress/` по `ticket_id` из контекста
2. **Определяет тип** задачи из frontmatter → загружает соответствующий воркфлоу
3. **Выполняет работу** по шагам воркфлоу
4. **Проверяет DoD** — все критерии готовности должны быть выполнены
5. **Записывает результат** в секцию Result тикета
6. **НЕ перемещает тикет** — это делает pipeline автоматически

## Как расширять

### Добавить новый воркфлоу

1. Создать файл в `workflows/` (например, `workflows/deploy.md`)
2. Добавить запись в таблицу маршрутизации в `SKILL.md`
3. Структура воркфлоу: Вход → Шаги → Антипаттерны → Выход

### Добавить knowledge-модуль

1. Создать файл в `knowledge/` (например, `knowledge/code-standards.md`)
2. Добавить запись в таблицу «Загрузка знаний» в `SKILL.md`
3. Добавить маркер расширения: `<!-- РАСШИРЕНИЕ: ... -->`

### Примеры расширений

- `knowledge/code-standards.md` — стандарты кода проекта
- `workflows/deploy.md` — воркфлоу для задач деплоя
- `knowledge/testing-guide.md` — руководство по тестированию
