# Воркфлоу: Research-задачи (RSH)

Специализированный воркфлоу для задач типа `rsh` — исследования с использованием внешних источников.

## Вход

- Тикет с `type: rsh` в `in-progress/`
- Тема и скоуп исследования из описания тикета

## Шаги

### 1. Read — понять тему и скоуп

1. Прочитать тикет, выделить:
   - Основную тему исследования
   - Конкретные вопросы
   - Ожидаемый формат результата
   - Путь для записи отчёта (обычно `reports/RSH-XXX-*.md`)
2. Прочитать SKILL.md соответствующего скила исследований для понимания методологии

### 2. Research — сбор данных

Вызвать скрипт исследований через bash:

```bash
node <путь-к-скрипту-из-SKILL.md-исследований> "исследовательский запрос"
```

Для сложных тем — несколько запросов с разными фокусами:

```bash
node <путь> "запрос 1"
node <путь> --model perplexity/sonar-pro "запрос 2"
```

**Доступные модели:**
- `perplexity/sonar-deep-research` — по умолчанию, глубокий ресерч (5-10 мин)
- `perplexity/sonar-pro` — быстрый с источниками (10-30 сек)
- `perplexity/sonar` — простые справки (5-15 сек)

### 3. Synthesize — анализ и верификация

1. Проанализировать stdout от скрипта
2. Верифицировать данные (кросс-проверка между источниками)
3. Структурировать результаты
4. Оформить финальный отчёт

### 4. Write — запись отчёта

Записать отчёт в путь из тикета.

## Антипаттерны

- Выдумывать данные без источников
- Использовать только собственные знания без вызова скрипта исследований
- Пропускать верификацию результатов
- Игнорировать указанный в тикете путь для отчёта

## Выход

- Отчёт записан в указанный путь
- Секция Result в тикете заполнена
- Все данные подкреплены источниками
