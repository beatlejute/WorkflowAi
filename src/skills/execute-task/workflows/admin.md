# Воркфлоу: Admin-задачи

Специализированный воркфлоу для задач типа `admin` — настройка, конфигурация, файловые операции.

## Вход

- Тикет с `type: admin` в `in-progress/`
- Список файлов в `context.files`

## Ключевое требование

Admin-задачи требуют **реального изменения файлов**. Нельзя завершить задачу без фактических правок.

## Шаги

### 1. Read — прочитать все файлы из контекста

```
Для каждого файла из context.files:
  → Read(файл)
  → Зафиксировать текущее состояние
```

### 2. Edit/Write — внести изменения

- Использовать инструмент Edit для точечных правок
- Использовать инструмент Write для создания новых файлов
- Каждое изменение должно соответствовать описанию задачи

### 3. Verify — перечитать и проверить

```
Для каждого изменённого файла:
  → Read(файл) — убедиться что изменения применены корректно
  → Сравнить с ожидаемым результатом
```

## Чеклист самопроверки

Перед завершением admin-задачи:

- [ ] Все файлы из `context.files` прочитаны?
- [ ] Изменения внесены инструментом Edit/Write (не просто описаны)?
- [ ] Изменённые файлы перечитаны и верифицированы?
- [ ] Секция Result содержит конкретный результат (не заглушку)?

## Антипаттерны

- Завершение задачи без реального изменения файлов
- Пустая или шаблонная секция Result
- Описание изменений только в Summary без фактических правок

## Выход

- Файлы изменены и верифицированы
- Секция Result заполнена с перечнем всех изменений
