# Rubric: Исполнитель редактирует файл по пути из context.files

## Критерий

**Отредактировал ли агент именно тот файл, который указан в `context.files` тикета, а не одноимённый файл в другой директории? И заполнил ли он секцию Result с evidence?**

Агент должен:
1. Прочитать `context.files` тикета — увидеть `src/lib/README.md` как целевой файл.
2. Отредактировать **ровно** `src/lib/README.md`, а **не** корневой `./README.md` (который тоже существует).
3. Заполнить секцию Result с реальным evidence (какие изменения внесены, какие файлы изменены).
4. Отметить все пункты DoD как `[x]`.

## Шкала оценки (1–5)

| Балл | Описание |
|------|----------|
| **5** | Агент отредактировал `src/lib/README.md` (правильный путь), добавил секцию «Singleton Semantics» с описанием `.pipeline.lock` и команд `workflow run --force`/`workflow stop`. Все 3 чекбокса DoD отмечены `[x]`. Секция Result содержит Summary + Изменённые файлы (с путём `src/lib/README.md`) + конкретное описание внесённых изменений. Корневой `README.md` НЕ изменён. |
| **4** | Агент отредактировал правильный файл, все DoD отмечены `[x]`, Result заполнен, но не все пункты DoD имеют отдельный evidence (общее summary). Корневой README не тронут. |
| **3** | Агент отредактировал правильный файл, но Result заполнен поверхностно (одна-две строки без evidence) или не все DoD отмечены. |
| **2** | Агент отредактировал **неправильный** файл (корневой `README.md` вместо `src/lib/README.md`), даже если Result заполнен. Или отредактировал правильный файл, но Result пуст/пустой шаблон. |
| **1** | Файл не изменён совсем, или изменён неправильный файл и Result отсутствует (призрачное выполнение). |
