# Воркфлоу: CHECK — Проверка актуальности тикета

Единственный воркфлоу скила. Определяет, актуален ли тикет для выполнения.

## Алгоритм выполнения

### 1. Загрузить алгоритм

Прочитай `algorithms/relevance-check.md` — основной алгоритм проверки (шаги, таблица решений, граничные случаи).

### 2. Загрузить knowledge (по необходимости)

| Модуль | Когда |
|--------|-------|
| `knowledge/ticket-states.md` | При проверке состояния тикета и зависимостей |
| `knowledge/plan-states.md` | При проверке родительского плана |
| `../shared/*` | При необходимости общей экспертизы — проверь индекс `../shared/README.md` |

### 3. Прочитать тикет

Путь: `.workflow/tickets/in-progress/{ticket_id}.md`

Извлечь из frontmatter: `parent_plan`, `dependencies`, `conditions`.
Извлечь из тела: DoD-чеклист, секцию `## Ревью`, секцию `## Блокировки`.

### 4. Выполнить проверки по алгоритму

Последовательно выполни шаги 0-6 из `algorithms/relevance-check.md`.
Первая сработавшая проверка → `irrelevant` (short-circuit).

### 5. Записать результат при irrelevant

Если вердикт `irrelevant` — выполни шаг 7 алгоритма (запись в таблицу ревью тикета).

### 6. Вывести результат

> ⛔ **ОБЯЗАТЕЛЬНО:** RESULT-блок должен быть в ответе ВСЕГДА. Без него пайплайн не может определить статус.

Сформируй блок `## Result` с таблицей проверенных критериев и машиночитаемый блок:

```markdown
## Result

| # | Критерий | Статус |
|---|----------|--------|
| 0 | Тикет не в skipped | ✓ / ✗ |
| 0 | Последнее ревью не failed | ✓ / ✗ |
| 2 | Тикет не заблокирован | ✓ / ✗ |
| 3 | Родительский план активен | ✓ / ✗ |
| 4 | DoD не выполнен | ✓ / ✗ |
| 5 | Зависимости активны | ✓ / ✗ |
```

```
---RESULT---
status: relevant | irrelevant
reason: <причина, если irrelevant или review_failed_needs_rework если relevant>
---RESULT---
```

## Чеклист завершения

- [ ] Алгоритм `relevance-check.md` загружен
- [ ] Тикет прочитан из `in-progress/`
- [ ] Все шаги алгоритма (0-6) выполнены последовательно
- [ ] При `irrelevant`: запись добавлена в таблицу ревью
- [ ] Машиночитаемый блок `---RESULT---` выведен
- [ ] Вердикт `reason` — из допустимого списка значений
