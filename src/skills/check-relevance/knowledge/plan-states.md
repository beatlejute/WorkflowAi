# Состояния планов и их семантика

## Расположение планов

| Директория | Семантика |
|------------|-----------|
| `.workflow/plans/current/` | Активные планы |
| `.workflow/plans/archive/` | Архивные планы |

## Состояния плана (frontmatter `status`)

| Статус | Семантика | Тикеты актуальны? |
|--------|-----------|-------------------|
| `active` | План в работе | Да |
| `draft` | Черновик, ещё не утверждён | Да (fail-safe) |
| `completed` | План полностью завершён | **Нет** — все тикеты плана неактуальны |
| `archived` | План заархивирован | **Нет** — все тикеты плана неактуальны |
| `cancelled` | План отменён | **Нет** — все тикеты плана неактуальны |

## Правила проверки

1. Путь к плану берётся из frontmatter тикета: поле `parent_plan`
2. Если `parent_plan` пуст или файл не найден → fallback `relevant` (fail-safe)
3. Статус плана читается из frontmatter: поле `status`
4. Неактивные статусы: `completed`, `archived`, `cancelled`

## Пример frontmatter плана

```yaml
---
id: PLAN-001
title: Пример плана
status: active
created_at: "2026-03-01"
---
```
