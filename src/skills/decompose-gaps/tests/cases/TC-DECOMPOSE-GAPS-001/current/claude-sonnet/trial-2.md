### Проверка scope (алгоритм scope-check.md)

**Шаг 2a — Проверка исключений из scope плана:**

Из плана извлечена секция исключений:
> *«Исключено из scope: исправление дефектов»*

Gap «Нужно исправить дефект маппинга полей» — это буквально «исправление дефекта». Он **прямо подпадает** под исключение из scope плана.

**Результат проверки:**

```
check_scope(gap) → {
  in_scope:     false,
  reason:       "Gap подпадает под исключение из scope плана: 'исправление дефектов'",
  check_failed: "2a"
}
```

Gap **отклонён на шаге 2a**. Дальнейшие проверки (3, 4) не применяются.
