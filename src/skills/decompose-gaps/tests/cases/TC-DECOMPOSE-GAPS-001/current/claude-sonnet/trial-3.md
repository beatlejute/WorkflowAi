## Проверка scope: Gap «Исправить дефект маппинга полей»

### Применение алгоритма scope-check

**Gap:** «Нужно исправить дефект маппинга полей»

**Шаг 1 — Проверка источника:**
Gap описывает конкретный дефект в выполненной работе → источник валидный (секция «Риски и проблемы» / «Выполненные задачи»). ✅

**Шаг 2 — Принадлежность к плану:**
Задача по маппингу полей присутствовала в плане. ✅

**Шаг 2a — Проверка секции «Исключено из scope»:**
План содержит явную запись:

> «Исключено из scope: **исправление дефектов**»

Gap «исправить дефект маппинга полей» непосредственно подпадает под данное исключение.

**Результат проверки 2a:**
```
in_scope: false
reason: "Gap подпадает под исключение из scope плана: 'исправление дефектов'"
check_failed: "2a"
```

**Дальнейшие шаги алгоритма:** не выполняются — gap отклонён на шаге 2a.
