**Анализ scope gap**

Gap: «Нужно исправить дефект маппинга полей».

Проверка по алгоритму `scope-check`:
1. **Проверка источника** – источник не указан, но допустим, что gap взят из валидной секции отчёта.
2. **Проверка принадлежности к плану** – план содержит явное исключение: «Исключено из scope: исправление дефектов». Описываемая работа (исправление дефекта) подпадает под это исключение.
3. **Проверка статуса плана** – статус плана неизвестен, но даже при `in_progress` исключение из scope остаётся в силе.
4. **Проверка workflow-управления** – gap не содержит инструкций по управлению workflow.

**Результат проверки:** gap **вне scope** декомпозиции.  
Причина: «Gap подпадает под исключение из scope плана: “исправление дефектов”».
