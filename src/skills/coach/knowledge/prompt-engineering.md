# Prompt Engineering для скилов

Принципы написания эффективных инструкций в SKILL.md и воркфлоу.

## Роль

| Принцип | Плохо | Хорошо |
|---------|-------|--------|
| Конкретность | «Ты — полезный помощник» | «Ты — Growth Marketing Lead в команде из 5 человек» |
| Границы | Только «Ты делаешь» | «Ты делаешь» + «Ты НЕ делаешь» |
| Контекст | Без контекста команды | С RACI-матрицей и связками ролей |

## Воркфлоу

1. **Глагол в начале** — «Определи», «Собери», «Проверь», не «Нужно определить»
2. **Ссылки на знания** — каждый шаг с данными ссылается на источник (`→ knowledge/module.md`)
3. **Чеклисты** — завершай воркфлоу проверочным списком
4. **Условия** — развилки описывай оба пути явно
5. **Нет «подумай»** — заменяй на конкретный метод/алгоритм

## Knowledge

1. **Факты, не мнения** — «CAC < LTV/3» вместо «CAC должен быть маленьким»
2. **Источники** — откуда данные
3. **Пороги числами** — «> 3x» вместо «значительно выше»
4. **Структура** — таблицы, категории, маркер `<!-- РАСШИРЕНИЕ -->`

## Algorithms

1. **Детерминированность** — одинаковый вход → одинаковый выход
2. **Числовые пороги** — «если score > 7» вместо «если оценка высокая»
3. **Граничные случаи** — что делать если данных нет
4. **Структура:** Вход → Алгоритм → Выход → Пример

## Антипаттерны

| Антипаттерн | Решение |
|-------------|---------|
| «Будь креативным» | Укажи конкретный метод |
| «Учти все факторы» | Перечисли факторы и веса |
| «При необходимости» | Укажи конкретное условие |
| Длинные абзацы | Таблицы, списки, чеклисты |
| Дублирование | Один источник правды + ссылки |
