# Эталонная анатомия скила

Справочник по структуре и содержанию идеального скила workflow-ai.

## Обязательная структура директорий

```
skill-name/
├── SKILL.md              # Ядро: роль, маршрутизация, принципы
├── README.md             # Документация для людей
├── workflows/            # Пошаговые алгоритмы по типам тикетов
├── knowledge/            # Экспертные знания домена
├── algorithms/           # Формализованные методы принятия решений
└── templates/            # Шаблоны вывода
```

## SKILL.md — обязательные секции

| Секция | Критерий качества |
|--------|-------------------|
| **Frontmatter** (name, description, ticket_prefix) | description 2-3 предложения, prefix уникален |
| **Роль** (делает / не делает) | Конкретно, без абстракций |
| **Маршрутизация** (тип → триггеры → воркфлоу) | Каждый тип → свой файл. **Только собственные типы** |
| **Загрузка знаний/алгоритмов/шаблонов** | Все файлы перечислены с условиями загрузки |
| **Принципы** (3-7) | Actionable, не декларативные |
| **Формат вывода** | Специфичен для домена |
| **Границы компетенции** | Перечислены смежные зоны с указанием куда направить |

## ⛔ Принцип изоляции скилов

Скилы НЕ знают о других скилах по имени. Используй: «любой скил проекта», generic ID (XXX-NNN).

**Изоляция от project-wide типов:** не хардкодить типы из `config.yaml:task_types`. Маршрутизация — по собственным типам скила, не по project-wide.

## Требования к качеству компонентов

| Компонент | Ключевые требования |
|-----------|--------------------|
| **Workflows** | Пронумерованные шаги, ссылки на knowledge/algorithms, чеклисты завершения |
| **Knowledge** | Таблицы/списки, маркер `<!-- РАСШИРЕНИЕ -->`, формулы явно, источники |
| **Algorithms** | Вход → алгоритм → выход, пороги числами, пример применения |
| **Templates** | Обязательные/опциональные поля, placeholder-подсказки, markdown |
| **README.md** | Название, структура, flow, как расширять |

## Shared Knowledge

При создании/аудите скила → загрузи `knowledge/shared-knowledge-guide.md` для правил межскиловых знаний.
