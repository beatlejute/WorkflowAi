### 3. Root Cause Analysis

**Физический автор ошибки:** скил `decompose-plan` (агент `claude-sonnet` на стадии `decompose-plan`).

**Цепочка причин:**
1. **Пропуск STOP‑GATE 9.B** в `workflows/decompose.md` — декомпозитор не получил параметр `id_ranges_json` (в логе контекст стадии содержит только `plan_id` и `plan_file`), но продолжил работу вместо остановки с `status: error`.
2. **Нарушение принципа дедупликации** — алгоритм `algorithms/deduplication.md` проверяет title/scope, но **не проверяет уникальность ID**; сканирование всех папок `tickets/` (включая `archive/`) должно было обнаружить существующие файлы `IMPL‑002.md` и `IMPL‑003.md`, но этого не произошло.
3. **Отсутствие пре‑валидации ID** перед записью файла — пост‑валидация (шаг 9.1, п.5) выполняется после сохранения, когда коллизия уже возникла.

**Семантический владелец:** скил `decompose-plan` полностью отвечает за генерацию уникальных ID тикетов. Инфраструктурная стадия `allocate-ticket-ids` (если она существует) не выполнилась или не передала диапазоны, что не освобождает декомпозитор от обязанности остановиться при отсутствии входного параметра.

### 4. Рекомендуемые правки скила `decompose-plan`

**Правка 1 (STOP‑GATE 9.B)** — усилить проверку наличия `id_ranges_json`:
- Добавить явную проверку в начале шага 9.B: «Если параметр отсутствует или пуст — немедленно остановиться, вывести `status: error`».
- Запретить любую fallback‑нумерацию (включая «начать с 001»).

**Правка 2 (пре‑валидация ID)** — добавить шаг 2.5 в раздел 9.1:
- Перед сохранением файла выполнить `Glob` по `.workflow/tickets/**/{TYPE}-{NNN}.md`.
- При обнаружении любого совпадения (включая `archive/`) — остановиться, сообщить об инфраструктурной ошибке.

**Правка 3 (дедупликация)** — дополнить `algorithms/deduplication.md` шагом 0:
- Если ID уже известен (после 9.B) — проверить уникальность через `Glob`; при коллизии — остановить дедупликацию.

**Универсальность правок:** все формулировки используют только универсальные термины (`PREFIX`, `NNN`, `tickets/`, `archive/`) без хардкода проектных префиксов или путей.
