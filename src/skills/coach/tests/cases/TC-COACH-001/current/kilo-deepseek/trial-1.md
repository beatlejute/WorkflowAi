### Выявленные проблемы

#### 1. **Неопределённый скил для stages скриптов** (HIGH)
- **Где:** stage `pick-first-task` (agent=`script-pick`), stage `check-conditions` (agent=`script-check-conditions`).
- **Симптом:** в логе указано `skill="undefined"`.
- **Риск:** отсутствие стандартизованных инструкций для этих этапов, возможные отклонения от принципов workflow.
- **Root cause:** конфигурация пайплайна не связывает агента со скилом.
- **Рекомендация:** создать или назначить скилы для скриптовых агентов, обеспечить загрузку инструкций.

#### 2. **Повторное использование ID тикетов между планами** (MEDIUM)
- **Примеры:** IMPL-002, IMPL-003 (архивированы из PLAN-002, созданы заново в PLAN-003).
- **Потенциальные проблемы:** семантическая путаница при ссылках на тикет; риск перезаписи файлов, если файловая структура не разделяет архив и бэклог.
- **Root cause:** скил `decompose-plan` не проверяет глобальную уникальность ID.
- **Рекомендация:** добавить проверку существования ID тикета (включая архив) перед созданием; при конфликте генерировать новый ID.

#### 3. **Отсутствие проверки уникальности ID при создании тикетов** (LOW)
- **Частный случай проблемы 2.** Акцент на процедуру декомпозиции.
- **Root cause:** алгоритм генерации ID, возможно, использует локальный счётчик.
- **Рекомендация:** использовать универсальный уникальный идентификатор (например, UUID) или комбинацию `plan_id` + инкремент.

#### 4. **Неполный лог пайплайна** (INFO)
- **Симптом:** лог обрывается на stage `check-conditions`, не показывает результат.
- **Не ошибка, но затрудняет анализ завершённости.**
- **Рекомендация:** убедиться, что пайплайн завершается корректно; при анализе запрашивать полный лог.
