# Спецификация формата alerts.jsonl

## Обзор

Файл `${stateDir}/alerts.jsonl` — это журнал системных событий (алертов) runner'а в формате [JSONL](https://jsonlines.org/) (один JSON-объект на строку). Файл используется для аудита, мониторинга и обработки автоматических событий workflow-ai.

## Классификация

Файл является **append-only**: новые записи добавляются только в конец файла, существующие записи **неизменяемы**. Это гарантирует целостность журнала и возможность аудита.

Кодировка файла — **UTF-8** (без BOM).

## Расположение

Путь: `${stateDir}/alerts.jsonl`

Где `stateDir` — directory состояния runner'а (настраивается конфигурацией).

## Структура записи (класс `ticket_auto_blocked`)

Каждая строка файла содержит JSON-объект с общими полями и специфичными для типа события.

### Общие поля (все записи)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `timestamp` | ISO8601 | Да | Время генерации события |
| `severity` | string | Да | Уровень критичности: `info`, `warning`, `error`, `critical` |
| `kind` | string | Да | Тип события/алерта (например: `ticket_auto_blocked`) |
| `project` | string | Да | ID проекта/workspace |
| `ticket_id` | string | Да | ID тикета, к которому относится событие |

### Специфичные поля для `ticket_auto_blocked`

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `attempts` | number | Да | Количество попыток до блокировки |
| `reason` | string | Да | Причина блокировки: `max_review_attempts`, `human_gate_rejected`, `human_gate_timeout`, `manual` |
| `stage` | string | Да | Стадия, на которой произошла блокировка (например: `review`, `qa`) |

## Пример записи

```json
{"timestamp":"2026-04-30T22:15:00.000Z","severity":"error","kind":"ticket_auto_blocked","project":"workflow-ai","ticket_id":"IMPL-042","attempts":3,"reason":"max_review_attempts","stage":"review"}
```

## Обработка файла

- JSONL-файл может обрабатываться по строкам (stream-парсинг) без загрузки всего файла в память.
- При чтении допускается пропуск повреждённых строк (invalid JSON) с логированием ошибки.
- Системы мониторинга могут фильтровать по `severity` и `kind`.

## Прогноз роста

Файл может достигать больших размеров (много строк) при активном использовании workflow-ai. Рекомендуется:
- **Ротация** (rotation) по размеру или времени (например, дневная ротация)
- **Архивация** старых файлов (сжатие gzip)
- Удаление/архивация записей старше N дней в соответствии с политикой хранения

## Безопасность

- файл может содержать конфиденциальные данные (ID тикетов, project ID); доступ должен быть ограничен
- Все записи должны иметь корректный формат ISO8601 в поле `timestamp` для корректной сортировки
