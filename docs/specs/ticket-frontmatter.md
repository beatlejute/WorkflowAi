# Спецификация полей frontmatter тикета

## Обзор

Frontmatter тикета — это YAML-блок в начале файла `.workflow/tickets/**/*.md`, содержащий метаданные задачи. Данная спецификация описывает стандартные поля, их типы и семантику, включая новые поля для автоматической блокировки.

## Общая структура

```yaml
---
id: IMPL-001
title: "Краткое описание задачи"
priority: 3
type: feature
created_at: "2026-04-30T10:00:00.000Z"
updated_at: "2026-04-30T12:00:00.000Z"
completed_at: ""
parent_plan: plans/current/PLAN-001.md
dependencies: []
conditions: []
context:
  files: []
  references: []
  notes: ""
complexity: medium
tags: []
---
```

## Стандартные поля

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `id` | string | Да | Уникальный идентификатор тикета (префикс-номер, например `IMPL-001`) |
| `title` | string | Да | Заголовок задачи |
| `priority` | number | Да | Приоритет (1 — highest, 5 — lowest) |
| `type` | string | Да | Тип задачи: `feature`, `bug`, `docs`, `refactor`, `test`, `chore` |
| `created_at` | ISO8601 | Да | Время создания тикета |
| `updated_at` | ISO8601 | Да | Время последнего обновления |
| `completed_at` | ISO8601 | Нет | Время завершения (заполняется пайплайном) |
| `parent_plan` | string | Нет | Путь к файлу родительского плана |
| `parent_task` | string | Нет | ID родительской задачи (если есть) |
| `dependencies` | array | Нет | Список ID зависимых тикетов |
| `conditions` | array | Нет | Условия запуска/перехода стадий |
| `context.files` | array | Нет | Список файлов, релевантных для задачи |
| `context.references` | array | Нет | Внешние ссылки (URL, документы) |
| `context.notes` | string | Нет | Дополнительные заметки от создателя |
| `complexity` | string | Нет | Оценка сложности: `trivial`, `low`, `medium`, `high`, `very_high` |
| `tags` | array | Нет | Теги для категоризации |

## Поля автоматической блокировки (auto_blocked_*)

Новые поля добавляются для фиксации случаев, когда тикет был автоматически заблокирован системой (workflow-ai) на human-gate стадиях (`review`, `qa`) из-за превышения лимитов попыток или таймаутов.

### `auto_blocked_reason`

Тип: `string` (enum)

Возможные значения:

| Значение | Описание |
|----------|----------|
| `max_review_attempts` | Превышено максимальное количество попыток на стадии review |
| `human_gate_rejected` | Human gate (стадия review/qa) вернул `rejected` максимальное количество раз |
| `human_gate_timeout` | Human gate не был обработан в срок (`expires_at` истёк) максимальное количество раз |
| `manual` | Тикет вручную заблокирован оператором/админом |

### `auto_blocked_attempts`

Тип: `number` (опциональный)

Количество попыток, которые были сделаны до автоматической блокировки. Используется для аналитики и понимания "нагруженности" тикета.

Пример: `3` — тикет прошёл 3 попытки на review/qa, затем был заблокирован.

### `auto_blocked_at`

Тип: ISO8601 строка (опциональный)

Временная метка, когда была применена автоматическая блокировка. Полезно для аудита и построения отчётов.

Пример: `"2026-04-30T22:15:00.000Z"`

## Backward compatibility

Поля `auto_blocked_*` являются **опциональными**. При парсинге frontmatter тикетов:

- Старые тикеты, созданные до внедрения данных полей, должны парситься **без ошибок**.
- Отсутствие полей `auto_blocked_*` интерпретируется как «автоблокировка не применялась».
- Эти поля заполняются только runner'ом и скриптами пайплайна при фактической блокировке.

## filled_at / auto_filled

> _Примечание: поля `filled_at` и `auto_filled` упомянуты в контексте задачи, но не являются частью финальной спецификации frontmatter тикета. Их использование ограничено внутренними механизмами заполнения данных._

## Пример frontmatter с auto_blocked полями

```yaml
---
id: IMPL-042
title: "Реализовать валидацию JSON-схемы для approval-файлов"
priority: 2
type: feature
created_at: "2026-04-30T08:00:00.000Z"
updated_at: "2026-04-30T14:00:00.000Z"
auto_blocked_reason: "max_review_attempts"
auto_blocked_attempts: 3
auto_blocked_at: "2026-04-30T22:15:00.000Z"
complexity: medium
tags:
  - api
  - validation
---
```
