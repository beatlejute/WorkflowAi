# pick-next-task.md

## Назначение скрипта
pick-next-task.js - Скрипт для выбора следующего тикета из директории ready/

## Usage
```
node pick-next-task.js
```

## Вывод результата
Скрипт выводит результат в формате:
```
---RESULT---
status: found
ticket_id: IMPL-001
---RESULT---
```

или если задач нет:
```
---RESULT---
status: empty
---RESULT---
```

## Новый статус: human_ready
Описывает ситуацию, когда в директории ready/ нет обычных (non-human) задач, готовых к выполнению, но есть задачи типа `human`, которые созрели для выполнения.

### Условие возникновения статуса human_ready:
- eligible non-human список пуст (нет обычных задач, удовлетворяющих зависимостям и условиям)
- Есть созревший human-тикет (удовлетворяющий зависимостям и условиям)

### Поля ответа при статусе human_ready:
```json
{
  "status": "human_ready",
  "ticket_id": "<ID тикета>",
  "priority": <приоритет тикета>,
  "title": "<название тикета>",
  "pending_count": <общее число созревших human-тикетов>
}
```

### Поле pending_count:
- Общее число созревших human-тикетов в директории ready/
- Всегда больше или равно 1 (≥ 1) когда статус равен human_ready
- Используется для manual-gate механизмов, чтобы показать, сколько human-задач ожидает ручного рассмотрения
