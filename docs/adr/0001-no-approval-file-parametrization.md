# ADR 0001: Не параметризация пути approval-файла в 1.3.0

## Статус

Принято

## Контекст

В текущей реализации `runner.mjs:1640-1642` жёстко вычисляет путь approval-файла как `{ticket_id}_{stageId}_{attempt}.json` с помощью функции `computeStepId(context, stageId, counters)`. Параметр `approval_file` в pipeline.yaml runner **не читает**.

Рассматривались альтернативы:
1. Параметризация пути через `approval_file` в конфиге
2. Сохранение детерминированного пути

## Решение

Выбран вариант сохранения детерминированного пути без параметризации. Причины:
- Существующая инфраструктура `manual-gate` в workflow-ai@1.2.0 уже использует жёстко вычисляемый путь
- Параметризация потребует изменений в `executeManualGate` и выходит за scope 1.3.0
- Детерминированный путь упрощает реализацию hook в `move-ticket.js`

## Последствия

1. `move-ticket.js` может искать approval-файлы по pattern `{ticket_id}_manual-gate-*_*.json` без необходимости знать конфиг pipeline
2. Пользователь не может настроить путь approval-файла — это отложено на будущие релизы
3. Упрощается тестирование и отладка — путь предсказуем
4. Сохраняется обратная совместимость с существующими реализациями `manual-gate`

Формат approval-файла остаётся стандартным: `{ticket_id}_{stageId}_{attempt}.json` в директории `.workflow/approvals/`.
